# LoadSprint — Гайд для владельца / админа (запуск и управление)

Это для того, кто **владеет** LoadSprint и управляет им. Простыми словами, по шагам.
Mac не нужен, всё делается на Windows.

---

## Где что находится

- **Живой сайт:** https://loadsprint.us.com (запасной: `loadsprint-production.up.railway.app`)
- **Хостинг (где крутится сайт):** Railway. Туда сайт попадает автоматически после
  `git push`.
- **Код на компьютере:** `C:\Users\vungu\Desktop\loadsprint`
- **Хранилище кода:** GitHub-репозиторий, привязанный к Railway.

---

## Как залить изменения на сайт (самое частое)

Когда тебе присылают обновлённый код (zip), действия такие:

1. **Распакуй** присланный архив. Внутри будет папка `loadsprint`.
2. Зайди в распакованную папку `loadsprint`, выдели **всё** (`Ctrl+A`), скопируй (`Ctrl+C`).
3. Открой свою рабочую папку `C:\Users\vungu\Desktop\loadsprint` и вставь (`Ctrl+V`).
4. Windows спросит про совпадающие файлы — **обязательно** выбери
   **«Заменить файлы в папке назначения»** и поставь галочку «Сделать это для всех».
   (Если выбрать «пропустить» — часть файлов не обновится и сборка может упасть.)
5. Открой терминал в этой папке и выполни три команды:
   ```
   git add -A
   git commit -m "update"
   git push
   ```
6. Railway сам подхватит изменения и пересоберёт сайт. Через пару минут обновление
   будет на https://loadsprint.us.com.

**Как понять, что всё хорошо:** в логах Railway будет «Compiled successfully» и деплой
дойдёт до конца. Если увидишь «Failed to compile» — чаще всего это значит, что файлы
заменились **не все** (вернись к пункту 4 и замени с перезаписью), либо пришли скриншот
ошибки — поправим.

> Важно: **никогда не добавляй файл `babel.config.js`** — он ломает сборку Next.js.

## Настройки сайта в Railway (переменные окружения)

Это «ключи и настройки», которые сайт читает при запуске. Их задают **в Railway**
(раздел Variables проекта), не в коде. Большинство уже стоит. Что это:

| Настройка | Зачем |
|---|---|
| `AUTH_SECRET` | Защищает вход (подпись сессий). Длинная случайная строка. |
| `ADMIN_EMAIL` / `ADMIN_PASSWORD` | Аккаунт админа (создаётся сам при первом входе). |
| `STRIPE_SECRET_KEY` / `STRIPE_WEBHOOK_SECRET` | Оплата через Stripe. |
| `RESEND_API_KEY` / `EMAIL_FROM` | Отправка писем-приглашений. |
| `HERE_API_KEY` | Карта/маршруты для фур, расстояние и время прибытия. |
| `ANTHROPIC_API_KEY` | ИИ: чтение rate confirmation и создание инвойсов. |
| `TWOFA_ENABLED` | `true` включает вход с кодом из письма (2FA). Сейчас выключено. |
| `SEED_DEMO` | `false` отключает демо-аккаунты. |
| `DATA_DIR` | Папка с данными (по умолчанию `data`). |

Если какой-то ключ не задан — связанная функция просто выключается, а сайт работает.

## Stripe (оплата) — что важно

- Тарифы (помесячные и «1 месяц»): Silver, Gold, Platinum — задаются в коде
  (`lib/billing-plans.ts`).
- Чтобы оплата корректно включала тариф, в Stripe-дашборде должен быть **вебхук**,
  указывающий на `https://loadsprint.us.com/api/billing/webhook`, и должен быть задан
  `STRIPE_WEBHOOK_SECRET`.
- Дополнительно сайт подтверждает оплату **при возврате** с Checkout, так что тариф и
  дата окончания ставятся сразу, даже если вебхук задержался.

## Админка `/admin`

1. Зайди под аккаунтом админа (email/пароль из настроек `ADMIN_*`).
2. Открой `https://loadsprint.us.com/admin`.

Что можно из админки:
- **Управлять аккаунтами** — видеть все аккаунты и их роли.
- **Выдать тариф** вручную — указать аккаунту план и **количество дней**
  (`0` = бессрочно, `>0` = действует столько дней; например 30).
- **Менять публичные цены** на странице тарифов.
- **Выдать инструмент «заморозка локации»** конкретному аккаунту.

> Подсказка: если у кого-то в меню «no expiry», но он оплачивал — можно просто выдать
> ему тариф на 30 дней из админки, и счётчик дней появится.

## Демо-аккаунты (для проверки)

Создаются автоматически (если не выключить через `SEED_DEMO=false`):
- Водитель: `demo.driver@loadsprint.us.com`
- Диспетчер: `demo.dispatch@loadsprint.us.com`

Удобно для проверки и для ревью Apple.

## Про данные

Данные (аккаунты, грузы, приглашения, настройки) хранятся **файлами** на сервере в
папке `DATA_DIR` (`data/`): `users.json`, `loads.json`, `invites.json`,
`notifications.json`, `settings.json`. На некоторых хостингах диск может очищаться при
деплое — поэтому демо-аккаунты и 2FA сделаны «самовосстанавливающимися». Для большого
роста этот файловый способ можно заменить на настоящую базу данных.

## Мобильное приложение

Приложение водителя (iPhone/iPad) — это **отдельный проект** на Expo (папка
`loadsprint-driver`), который подключается к этому же сайту. У него своя инструкция по
сборке. Правки на **сайте** не требуют пересборки приложения — приложение пересобирают
только если меняется что-то **внутри самого приложения**.

---

## Если что-то сломалось — куда смотреть

1. **Сайт не обновился** → проверь, что сделал `git push`, и посмотри логи в Railway.
2. **«Failed to compile»** → почти всегда файлы заменились не все; замени архив с
   перезаписью и снова `git push`. Если не помогло — пришли скриншот ошибки.
3. **Оплата не включает тариф** → проверь вебхук Stripe и `STRIPE_WEBHOOK_SECRET`.
4. **Письма не уходят** → проверь `RESEND_API_KEY` и `EMAIL_FROM`.

Сомневаешься — лучше спроси перед действием, чем сделай наугад.
